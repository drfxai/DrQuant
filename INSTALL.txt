============================================================================
 DrFX Quantum - Setup & Installation Guide
============================================================================

DrFX Quantum is a Telegram-style trading communication platform
(Node.js + PostgreSQL + Nginx + PM2), installed by the top-level install.sh.


============================================================================
 0. PREREQUISITES
============================================================================

  - A fresh Ubuntu/Debian server (tested on Ubuntu 24.04) with root access.
  - A domain name with an A record pointing at the server's public IP
    (required for HTTPS / Let's Encrypt).

IMPORTANT - how the code reaches your server:
  The installer runs from a `git clone` of this repository. Anything that lives
  only on a local machine reaches the server ONLY after it is committed and
  pushed to GitHub. After pushing, run `git pull` (or re-clone) on the server
  so it has the latest installer before running it.


============================================================================
 1. INSTALL (one command)
============================================================================

  git clone https://github.com/drfxai/DrFXQuant.git
  cd DrFXQuant
  sudo bash install.sh

install.sh is a one-shot installer. It will:
  - Install Node.js 20, PostgreSQL, Nginx, PM2, and (optionally) certbot.
  - Create the `drfx_quantum` PostgreSQL database and a `drfx` user.
  - Copy the application to /var/www/drfx-quantum (including the routes,
    public assets, middleware, services, realtime, and migrations).
  - Generate /var/www/drfx-quantum/.env with secure random secrets
    (JWT secret, TradingView webhook secret, etc.).
  - Create the base database tables, then apply every migration in
    migrations/*.sql (this creates the TradingView signal/webhook tables).
  - Start the app under PM2 behind an Nginx reverse proxy.
  - Optionally obtain an HTTPS certificate.

Prompts it will ask:
  - Domain (e.g. app.yourdomain.com)
  - Admin email
  - Admin password (min 6 chars, asked twice)
  - OpenRouter API key            (optional - press Enter to skip)
  - NowPayments API key           (optional - press Enter to skip)
  - NowPayments IPN secret        (optional - press Enter to skip)
  - Proceed? (y/n)
  - Setup SSL now? (y/n)

When it finishes:
  - The app is reachable at http://your-domain  (https if you enabled SSL).
  - Log in with the admin email and password you entered.

Post-install steps for the trading features:
  - The TradingView webhook shared secret was auto-generated. Find it in:
        /var/www/drfx-quantum/.env   (TRADINGVIEW_WEBHOOK_SECRET=...)
    Use that same secret in your TradingView alert payloads.
  - Create a channel named 'signals' inside the app so incoming webhook
    signals have a channel to broadcast to.
  - To enable the AI assistant, set OPENROUTER_API_KEY in the .env (re-run is
    not needed; just edit and restart). To enable crypto payments, set the
    NOWPAYMENTS_* values.

Everyday management:
  pm2 logs drfx-quantum            # view live logs
  pm2 restart drfx-quantum         # restart after editing .env
  pm2 status                       # process status
  nano /var/www/drfx-quantum/.env  # edit configuration
  sudo bash uninstall.sh           # remove the app


============================================================================
 2. TROUBLESHOOTING
============================================================================

App won't start / 502 Bad Gateway:
  pm2 status                       # is the process online or crash-looping?
  pm2 logs drfx-quantum --lines 50 --nostream
  curl -i http://127.0.0.1:3000/   # does the backend answer locally?
  - "Cannot find module './realtime/...'" -> the realtime/ directory is missing
    from /var/www/drfx-quantum. Re-run install.sh (it copies realtime/ and now
    fails fast if it is absent), or copy the folder in from a fresh clone.
  - "ECONNREFUSED 127.0.0.1:5432" / DB auth errors -> PostgreSQL isn't running
    or DATABASE_URL in .env is wrong.

TradingView webhook returns 500:
  - Ensure migrations were applied (install.sh "Step 3b"); the signals and
    webhook_logs tables must exist.

Login / admin not working:
  - The admin user is created/updated from ADMIN_EMAIL / ADMIN_PASSWORD in
    /var/www/drfx-quantum/.env on each boot.

PostgreSQL:
  sudo -u postgres psql -d drfx_quantum -c "\dt"     # list tables


============================================================================
 3. UPDATING
============================================================================

  cd /path/to/DrFXQuant
  git pull
  sudo bash install.sh        # idempotent: safe to re-run; re-applies
                              # migrations and refreshes the deployed copy


============================================================================
 4. FILE / PATH REFERENCE
============================================================================

  Install dir      /var/www/drfx-quantum
  Config           /var/www/drfx-quantum/.env            (chmod 600)
  Process manager  PM2 process name: drfx-quantum
  Web server       Nginx, proxying to 127.0.0.1:3000
  Database         PostgreSQL db "drfx_quantum", user "drfx"
